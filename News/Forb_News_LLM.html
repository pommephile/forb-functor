When Perfect Grammar Kills Language: Why Forb-Constrained LLMs Collapse

An experiment that was designed to succeed failed spectacularly — and in doing so, proved the theory it was testing. Token-level forbidden-pattern constraints make language models 79.7% worse. The fix requires rethinking what "constraint" means.
June 25, 2026

Suppose you wanted to make a language model perfectly grammatical. You take a corpus, enumerate every pair of words (or subword tokens) that never appear together, and forbid the model from generating those pairs. Intuitively, this should produce cleaner text. It does the opposite. The model collapses into incoherent repetition — "20², 20², 20², …" — with a constraint-violation rate higher than unconstrained generation. This is not a bug. It is a proof.

"99.6% of subword bigrams are forbidden. That leaves 0.4% of the combinatorial space. The model has nowhere to go. The collapse is mathematically inevitable."

The Experiment

UOTA implemented Forb-constrained decoding on a real large language model: Llama-3.2-3B running on an Oracle Cloud ARM instance, with the WikiText-2 corpus (10.8 million characters). The vocabulary has 2,892 subword tokens. The Forb functor was applied at the token level: count all token bigrams in the training corpus, mark those with count zero as forbidden, and mask them from the model's output distribution at every generation step.

The result was catastrophic:

Decoding modeConstraint violation rateOutput quality
Unconstrained76.6% (baseline)Coherent English
Forb-constrained (token level)79.7%Incoherent: "20², 20², 20², …"
Forb-constrained (word level, 1,046 words)0%Coherent, stylistically constrained

Token-level Forb made the model worse at following constraints than having no constraints at all. The reason is combinatorial: 2,892 tokens yield 2,892² = 8.36 million possible bigrams. Only 0.37% are observed in the training corpus. 99.63% are forbidden. The model, forced to choose only from the 0.37%, runs out of valid continuations within a few tokens and falls into whatever repetitive loop survives the mask.

The Phase Transition

The counter-experiment is crucial. When the same Forb pipeline is applied at the word level — using Alice in Wonderland, 1,046 unique words, 1.10% bigram observation rate — the constraint violation rate is zero percent. The output is coherent and stylistically constrained to the source text's vocabulary patterns. There is a qualitative phase transition between word-level and token-level Forb constraints:

The phase boundary. When forbidden rate exceeds ~95%, the constrained generation space becomes path-disconnected: from any valid token, almost no valid next token exists, and the few that do lead only to other dead ends. The transition from coherent (word-level, 98.9% allowed) to collapsed (token-level, 0.37% allowed) is not gradual — it is a percolation threshold in the bigram graph.

Feature Compression: The Necessary Fix

The failure is not a refutation of Forb — it is a validation of the theory's core claim. Forb's foundational Theorem 4.1 requires a feature map F: S → {0,1}ⁿ that compresses the symbol space before constraints are applied. The token-level experiment deliberately skipped this step, applying Forb directly to the raw token space. The resulting collapse proves that feature compression is not optional — it is mathematically necessary.

UOTA demonstrates the solution: an 8-bit orthographic feature encoding (starts-with-space, uppercase, lowercase, numeric, punctuation, vowel, length bucket) that compresses the 2,892-token vocabulary to at most 2⁸ = 256 feature classes. The forbidden rate drops from 99.6% to approximately 60% — still constraining, but leaving the model enough combinatorial room to generate coherent text. The compression ratio is 868× (from 8.36 million raw bigrams to 256² = 65,536 feature bigrams).

Scope note. The 8-bit feature encoding is a proof-of-concept. Optimal feature design for LLM constrained decoding — balancing constraint strength against generation diversity — is an open problem. The feature-compressed Forb decoder has been implemented (scripts/llama_forb_feature.py) but the full end-to-end experiment with feature-level constraints on a real LLM remains to be run.
